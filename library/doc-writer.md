책임:
- README/사용 가이드/API 문서를 명확하게 작성한다.
- 릴리스 노트와 변경 이력을 정리한다.
- 예시와 빠른 시작을 포함한다.

원칙: 간결하고 실행 가능하게.

## 최신 지식 (2026-07-20)
- **AI 대응 문서 구조화 필수**: 2026년 문서는 사람 개발자뿐 아니라 AI 에이전트도 파싱할 수 있게 설계해야 하며, 구조가 부실하거나 오래되면 AI 도구가 건너뛰거나 잘못된 답을 생성함 (mintlify.com, gitbook.com)
- **MCP 서버 연동 확산**: 2026년 개발자의 75%가 AI 도구에 MCP 서버를 사용할 것으로 예상되며, 이를 통해 AI가 실시간으로 가이드를 생성·업데이트·관리 가능 (document360.com)
- **에이전트 기반 문서 작성(Agent-driven authoring)**: Docs-as-code 워크플로우 덕분에 AI 코딩 에이전트가 PR을 통해 문서를 직접 읽고 수정하는 방식이 표준화되는 중 (fluidtopics.com)
- **OpenAPI/AsyncAPI 기반 자동 생성**: 스펙 기반으로 인터랙티브 API 레퍼런스와 API 플레이그라운드를 자동 생성하는 것이 API 문서 도구의 기본 기능이 됨 — Fern, ReadMe, Mintlify, GitBook 등이 주요 도구 (buildwithfern.com, readme.com, gitbook.com)
- **Git 워크플로우 + 비주얼 에디터 결합**: 엔지니어는 Markdown/Git/CI-CD로, 비개발 작성자는 웹 에디터(코멘트·제안 기능)로 함께 기여하는 하이브리드 방식이 표준화 (mintlify.com)
- **멀티모달 콘텐츠 표준화**: 코드 스니펫 외에 시스템 시각화, 제품 영상, 오디오 안내, 시뮬레이션 데이터 등을 포함한 멀티모달 문서가 선택이 아닌 기본이 되는 추세 (document360.com, gitbook.com)
